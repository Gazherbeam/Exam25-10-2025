Tailwind CSS ‚Äî Quick Guide & Custom CSS

Quickstart (Play CDN)

For quick prototypes include the CDN script (not recommended for production):

<script src="https://cdn.tailwindcss.com"></script>

Then use utility classes anywhere:

Resize the window to see responsive classes at work.

Recommended: Build with Tailwind (npm + PostCSS)

Steps to set up a production-ready build:

# initialize and install
npm init -y
npm install -D tailwindcss postcss autoprefixer

# create config and input/output
npx tailwindcss init -p
# create src/input.css (see below), then build:
npx tailwindcss -i ./src/input.css -o ./dist/output.css --watch

Contents of src/input.css:

@tailwind base;
@tailwind components;
@tailwind utilities;

/* Your custom layers can go here */
@layer base {
    /* small base reset or typography tweaks */
    body { @apply bg-slate-50 text-slate-900; }
}

@layer components {
    .btn {
        @apply inline-flex items-center px-4 py-2 rounded-md font-medium bg-indigo-600 text-white hover:bg-indigo-700;
    }
}

Sample tailwind.config.js (extend theme and purge content):

module.exports = {
    content: [
        "./src/**/*.{html,js}",
        "./public/index.html",
    ],
    theme: {
        extend: {
            colors: {
                brand: {
                    DEFAULT: '#1e3a8a',
                    light: '#3b82f6'
                }
            }
        }
    },
    plugins: [],
}

Custom CSS techniques

  • @apply ‚Äî compose utilities into a custom class (example: .btn above).
  • @layer base/components/utilities ‚Äî control where your custom rules sit relative to Tailwind's generated CSS.
  • tailwind.config.js ‚Äî add colors, spacing, fonts, or enable plugins and custom variants.
  • Purge/content ‚Äî list files where classes appear so unused utilities are removed in production builds.

Example: custom component with responsive and state variants

@layer components {
    .card {
        @apply p-6 rounded-lg bg-white shadow hover:shadow-lg transition;
    }
    .card-title {
        @apply text-lg font-semibold mb-2;
    }
    .card-desc {
        @apply text-sm text-slate-600;
    }
}

Tips & Best Practices

  • Prefer utility classes for layout and small patterns; create named components for reusable UI (buttons, cards).
  • Keep the content array in config accurate so tree-shaking removes unused CSS.
  • Use design tokens (extend theme) instead of hard-coded values in @apply to remain consistent.
  • For large projects, consider component extraction (e.g., as React/Vue components) and design system documentation.

Short Reference

Responsive: prefix with sm:, md:, lg:, etc.

State: hover:, focus:, active:, disabled:.

Pseudo-class: dark: (if using dark mode), custom variants via plugins are possible.

Use the CDN for quick demos. For production, build Tailwind with PostCSS and configure content to purge unused classes.
